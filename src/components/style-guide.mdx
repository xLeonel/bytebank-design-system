### Uso recomendado

- `--bb-primary` — ações principais, headers de card/modal, estado ativo da sidebar.
- `--bb-secondary` — ações secundárias em outline (borda + texto, sem preenchimento).
- `--bb-accent` — destaques, links secundários e elementos de ênfase.
- `--bb-success` — confirmações, valores positivos e aprovações.
- `--bb-warning` — avisos suaves; usado também em texto sobre fundos escuros (4.4:1).
- `--bb-error` — erros, valores negativos e ações destrutivas.
- `--bb-dark` — textos principais e títulos em fundos claros.

## Tipografia

Fonte padrão do design system: **Inter** (Google Fonts).

> **Atenção para quem consome a lib:** a Inter não é carregada automaticamente. Adicione-a no seu projeto por uma das opções abaixo.

**Opção 1 — Google Fonts (HTML)**
```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@300;400;500;600;700;800&display=swap" rel="stylesheet">
```

**Opção 2 — Fontsource (npm, sem dependência externa)**
```bash
npm install @fontsource/inter
```
```ts
// No entry point do projeto (ex: main.ts / _app.tsx)
import '@fontsource/inter/300.css';
import '@fontsource/inter/400.css';
import '@fontsource/inter/500.css';
import '@fontsource/inter/600.css';
import '@fontsource/inter/700.css';
import '@fontsource/inter/800.css';
```

## Acessibilidade

- Contraste mínimo **4.5:1** para texto normal (WCAG AA) e **3:1** para texto grande (≥ 24px).
- Use `aria-label` e textos descritivos em todos os elementos interativos.
- Garanta foco visível em botões, campos e links.
- Prefira texto legível e espaçamento confortável entre elementos.
